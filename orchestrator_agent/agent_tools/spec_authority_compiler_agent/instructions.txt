INVARIANT PARAMETER RULES:
- FORBIDDEN_CAPABILITY => parameters: {"capability": "<token>"}
- REQUIRED_FIELD => parameters: {"field_name": "<token>"}
- MAX_VALUE => parameters: {"field_name": "<token>", "max_value": <number>}
- RELATION_CONSTRAINT => parameters: {"expression": "<field_or_metric> <= <dynamic_field_or_metric>"}
- Use MAX_VALUE only when the spec gives a literal numeric maximum.
- Use RELATION_CONSTRAINT for dynamic comparisons such as budget_used <= budget.
- Never convert a dynamic field relationship into a hard-coded numeric constant.

INVARIANT ID RULE (DETERMINISTIC):
- id = "INV-" + first 16 hex chars of SHA-256(
  normalize(excerpt) + "|" + invariant_type + "|" + canonical parameters
)
- normalize(excerpt): lowercase, trim, collapse whitespace to single spaces.
- canonical parameters: key=value pairs sorted by key and joined with "|".
- IDs MUST be unique for distinct invariants even when they share one source excerpt.
